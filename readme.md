## Módulo 2 - Levantamento de Requisitos (Continuação)

- O que é Levantamento de Requisitos
    - Transformar as necessidades do negócio em informações técnicas

- Requisitos Funcionais e Não Funcionais
    - RF - O que o sistema vai fazer
    - RNF - Como o sistema vai fazer

- Requistos, Regras, Restrições
    - Requistos: A Parte Técnica do sistema
    - Regras: As Necessidade dos Clientes
    - Restrições: Limitações impostas pela, línguagem, leís, normas, 

- Modelo de Documentação de Requisitos

|Campo|Conteúdo|Exemplo|
|----|------|------|
| ID | Código | RF-001 |
| Título | Valor | Tela de Login |
| Descrição | O Ator deve ser Capaz de | O Cliente conseguirá fazer Login | 
| Prioridade | Valor da Prioridade | Alta |
| Regras Vinculadas | Outras Regras Relacionada | RN-001 - Banco de Dados |


### Técnicas de Levantamento de Requisitos

- Brienfing: Coleta de Dados Iniciais
    - Nome do Projeto
    - Objetivos do projeto (Levantamento de Requisitos)
    - Público-Alvo
    - Orçamento
    - Prazos
    - Elementos de Design (Cores, Fontes, Estilos, Logomarca, Logotipo, Slogan)
        - Manual de Identidade Visual 
    - Identidade do Cliente
        - Missão, Visão, Valores

- BrainStorm ou BrainStroming: Técnica de Sugestão de Ideais para Solução de Problemas
    - Estrturadas: Reuniões pautadas, com ordem de fala e tempo pré-determinado
    - Não Estrturadas: apenas os probelmas são apresentados
    - Mistas: misturam técnicas de reuniões estrturadas com não estrturadas, podem ter tempo de fala, mas sem ordem de fala
    - Ferramentas para BrainStorm:
        - Miro, Mentimeter, OneNote, Google Docs
